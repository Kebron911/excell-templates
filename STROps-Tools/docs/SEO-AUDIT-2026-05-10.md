# strops.tools — SEO Audit

**Audit date:** 2026-05-10
**Auditor:** Claude (Opus 4.7)
**Scope:** Full source-grounded audit of `STROps-Tools/` (Astro static + React islands + Tailwind).
**Live URL:** https://strops.tools
**Production sitemap fetched:** yes (https://strops.tools/sitemap-index.xml — 104 URLs in `sitemap-0.xml`)

## Executive summary

The site has a strong-out-of-the-gate SEO foundation. Per-page title / meta description / canonical / OG tags ship via a single `Layout.astro`. JSON-LD Organization + WebApplication + FAQPage on calculators; HowTo + FAQPage on every `/maintenance/[task]/` and `/replace/[item]/` programmatic page; BlogPosting + BreadcrumbList on blog posts. Per-page Satori OG images exist for 100+ slugs (`public/og/`). RSS feed lives at `/feed.xml` with 7 posts. 104 indexable URLs covering 7 calculators + 30 maintenance pages + 51 replacement-cost pages + 7 blog posts + 3 lead magnets + 4 statics. The work that already exists is ahead of where strhost.tools was at the same point in its lifecycle.

The biggest, fixable problems:

1. **`/feed.xml` is missing from the sitemap.** Live `sitemap-0.xml` has 104 URLs but does NOT include `/feed.xml`. Same bug class as strhost.tools' P0. Easy fix in `astro.config.mjs:14-19` via sitemap `customPages`. (P1 — feed is auto-discoverable on blog posts already, but only on blog posts.)
2. **No `/404` page exists.** No `src/pages/404.astro` in the tree. Live `https://strops.tools/this-page-does-not-exist-test-404` returns 404 with no branded body — the host's generic page. Bounced traffic and crawlers see nothing. (P0)
3. **No RSS autodiscovery `<link rel="alternate">` on most pages.** Only blog post pages emit it (verified on `/blog/airbnb-turnover-gap/` and `/blog/`). Home, tools, magnets, programmatic pages all lack the `<link rel="alternate" type="application/rss+xml">` in `<head>`. (P1)
4. **Sitemap vs canonical trailing-slash mismatch.** Sitemap entries use trailing slash (`https://strops.tools/about/`), canonicals do not (`https://strops.tools/about`). Identical bug to strhost.tools. Live verification: every page audited shows this. Google may treat as duplicates and pick one. (P1)
5. **Programmatic thin-content risk on 46 of 51 `/replace/[item]/` pages and 25 of 30 `/maintenance/[task]/` pages.** Only 5 items each have narrative MDX in `src/content/replacement/` and `src/content/maintenance/`. The remaining 71 pages render boilerplate from `src/data/items.json` and `tasks.json`. Same failure mode as strhost's lodging-tax. Major risk for cluster suppression. (P0)
6. **Contact form action is unset.** `/contact` body literally reads "replace the action URL with your Formspree (or equivalent) endpoint when wiring goes live." Form ships with no backend. Submissions go nowhere. (P0)
7. **No `BreadcrumbList` schema on programmatic clusters or static pages.** `breadcrumbJsonLd` helper exists at `src/lib/seo.ts:96` and is wired on blog posts only. Would benefit `/replace/[item]/` (`Home -> Replacement -> Queen Mattress`) and `/maintenance/[task]/` significantly. (P1)
8. **`Organization.sameAs[]` is sparse.** `src/lib/seo.ts:9` only lists `thestrledger.com`. No social profiles, no sister-site cross-links. Google uses `sameAs` for entity disambiguation. (P2)
9. **No font preload, no `font-display: swap` declaration in head.** `Layout.astro:39-87` head emits no `<link rel="preload">` for fonts. CLS / FOUT risk. (P2)
10. **Calculator hydration directive unverifiable from live HTML.** No `client:load` or `client:visible` markers appear in the rendered HTML — Astro strips them at build. Source: `turnover-scheduler.astro:33` uses `client:load`. Same `client:load` pattern across all 7 calculators (verified with grep on `src/pages/`). Should be `client:visible` for below-the-fold islands. (P1)

Headline score: **76/100** — strong base, very fixable to **88+** within a sprint.

## Prioritized remediation roadmap

### Quick wins (≤1 hour each)

1. **Create `src/pages/404.astro`** with Layout + cross-links to all 7 tools + 3 magnets + blog + programmatic indexes. *Recovers bounced traffic; eliminates soft-404 risk.*
2. **Wire the contact form** to Formspree/Basin or replace with a `mailto:hello@strops.tools` fallback. *Closes a P0 funnel leak.*
3. **Add `/feed.xml` to sitemap `customPages`** in `astro.config.mjs:14-19`. *Recovers RSS discoverability.*
4. **Decide trailing-slash convention** and fix `Layout.astro:34` canonical to match the sitemap. *Removes a duplicate-content signal.*
5. **Move `<link rel="alternate" type="application/rss+xml">` to `Layout.astro` head unconditionally.** *RSS discoverable from every entry point.*
6. **Switch `client:load` → `client:visible`** across all 7 calculator pages. *Real LCP/INP win for mobile.*
7. **Populate `Organization.sameAs[]`** in `src/lib/seo.ts:9` with sister sites. *Entity disambiguation.*
8. **Update H1 on 6 tools** to include "Airbnb" qualifier matching `<title>`. *Title↔H1 alignment.*

### Medium (≤1 day each)

9. **Emit `BreadcrumbList`** on programmatic + static pages via the existing `breadcrumbJsonLd` helper. *Schema coverage.*
10. **Emit `ItemList` schema** on `/replace/` and `/maintenance/` index pages. *List-style SERP features.*
11. **Add `Organization` + `ContactPage`/`AboutPage`/lead-magnet schemas** to the 5 pages that currently emit nothing.
12. **Audit `src/styles/global.css`** for fonts, add `font-display: swap` + preload critical weights.

### Strategic (≥1 day each)

13. **Write 25 narrative MDX for `/replace/[item]/` + 25 for `/maintenance/[task]/`.** Single biggest SEO unlock for the site. ~50 files, ~1 day of writing or 4 hours with a tight LLM prompt + edit pass. (See `KEYWORD-RESEARCH-2026-05-10.md` §4.)
14. **Affiliate placements** — verify `affiliates.json` vendor matches per tool. `turnover-scheduler.astro:48` already renders an `AffiliateCard`; audit the other 6 tools.
15. **GA4 + Search Console wired into deploy pipeline.** GA4 hook in `Layout.astro:66-86` is gated on `PUBLIC_GA4_ID`; confirm set in production CI/CD. (Layout already has cross-domain linker config — good.)
16. **Build out blog cadence to 12+ posts.** 7 posts is OK; 12+ before the next audit creates topical authority compounding.
