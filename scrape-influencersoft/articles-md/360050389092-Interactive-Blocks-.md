## What are the interactive blocks for?

Interactive blocks are used in [a constructor when you create or edit pages](https://help.influencersoft.com/hc/en-us/articles/360050388752-Creating-and-Editing-Pages-in-the-Page-Builder). These blocks could be easily hidden by one click with no need to refresh a page.

Let us look at the following situation: you run an online webinar and want the button “Order now” pop up once you discuss your offer. But the webinar is live, so it is hardly possible to predict when this happens. Or you want to show this button several times for short periods of time at the moments when the participants are prepared for it the best. How could this be done? For example, you can edit the page during the webinar, or you can choose other alternatives, but they all have their own disadvantages: no block will appear (and disappear) on the page which has already been opened by a user unless the user refreshes it. This very problem is solved by the interactive blocks. They are typical [widgets](https://help.influencersoft.com/hc/en-us/articles/360050388752-Creating-and-Editing-Pages-in-the-Page-Builder#h_01ERE5PQWCMBVRMZG11TXZAN39) we get used to: a text, forms, buttons, images, timer — any widget could be interactive. This is true for whole [sections](https://help.influencersoft.com/hc/en-us/articles/360050388752-Creating-and-Editing-Pages-in-the-Page-Builder#h_01ERE5PHNM8CWKN9GAFXADPPYD): you can show and hide them if necessary.

## How to handle interactive blocks?

### Enabling the function

The widget becomes interactive once it is [added to the page](https://help.influencersoft.com/hc/en-us/articles/360050388752-Creating-and-Editing-Pages-in-the-Page-Builder#h_01ERE5PQWCMBVRMZG11TXZAN39) in the template constructor, and the option is ticked in the widget settings. See below:

 ![blobid1.png](https://help.influencersoft.com/hc/article_attachments/360072496391/blobid1.png)

A similar process can be followed for the sections. Click “Settings” in the upper left corner of the section and tick interactivity.

 ![blobid2.png](https://help.influencersoft.com/hc/article_attachments/360072272312/blobid2.png)

Do not forget to save the page prior to exiting.

### Changing the status

Now your page has at least one interactive block, and the constructor has a new switch button:

 ![blobid3.png](https://help.influencersoft.com/hc/article_attachments/360072496411/blobid3.png)

If you click the button, you enter enable and disable mode for the blocks which you have ticked earlier. You can enter the mode without opening the page in the constructor; a new button will appear in the general view of editing the page.

 ![blobid4.png](https://help.influencersoft.com/hc/article_attachments/360072496431/blobid4.png)

However, you can do that even quicker! The section “Website — Pages” displays a list of all pages when you open the site you need (if it is just one site, it will immediately open). If the page has interactive blocks, then the “enable/disable a page” switch button will be next to the “interactivity mixer” button:

 ![blobid5.png](https://help.influencersoft.com/hc/article_attachments/360072496451/blobid5.png)

You can choose any way to enter the editor where you can enable or disable the blocks by one click. Each has its own switch button.

 ![blobid6.png](https://help.influencersoft.com/hc/article_attachments/360072272332/blobid6.png)

![blobid7.png](https://help.influencersoft.com/hc/article_attachments/360072272352/blobid7.png)

The page is automatically saved after each switch, and the changes could be seen live by all page users. No extra page save is needed. The save button at the bottom of the page is just part of the page’s template and has no use. Note: if a user opens a page with no interactive blocks yet, they will appear or disappear once the page is refreshed. In other words, if you conduct a webinar and decide to make some blocks interactive, this will not work (with no page reload on the users’ side). However, if you have made necessary preparations and made the required bock interactive, then they will be enabled without the need to refresh the page during the webinar.
