# CREDENTIALS — Single source of truth for "where does X live?"

> **All values live in ONE file: `<repo-root>/.env`** (gitignored).
> Per-project `.env.local` files are auto-generated by `pnpm sync:env`.
> Read [`docs/env-setup.md`](docs/env-setup.md) for the full design.
>
> **Vaultwarden = off-disk disaster-recovery copy.** This file documents
> which Vaultwarden entry holds what. Secrets themselves never appear here.

---

## Quick audit

```bash
# Which keys are actually set (no values printed)?
pnpm sync:env --print
```

```bash
# Are per-project files in sync with root .env? (CI)
pnpm sync:env:check
```

## CI / GitHub Actions

Deploy workflows do NOT read root `.env` (not committed). Each workflow
reads from GitHub Actions secrets and writes a temporary `.env` at build:

- `STR_SSH_KEY` — Hostinger SSH key (all deploys)
- `LISTINGAUDIT_GA4_ID`, `LISTINGAUDIT_ADMIN_TOKEN`, `LISTINGAUDIT_IP_HASH_SALT` — written to a build-time `.env`
- Same pattern for any other CI-required key

When adding a key that also needs CI: add it to root `.env` AND to GitHub
Actions secrets. Two consumers, two places.

## n8n Docker host — SSH details

> Enables operations the n8n REST API cannot do: install community nodes,
> edit `N8N_*` env vars, tail container logs, modify Traefik labels,
> restart services, back up the data volume.

```
N8N_SSH_HOST          = 192.168.86.25
N8N_SSH_PORT          = 22
N8N_SSH_USER          = n8n-ops
N8N_SSH_KEY_PATH      = C:\Users\Kebron\.ssh\n8n-ops_ed25519

N8N_COMPOSE_DIR       = /home/kebron/git/mydocker
N8N_SERVICE_NAME      = n8n
N8N_CONTAINER_NAME    = n8n
N8N_DATA_VOLUME       = /home/kebron/git/mydocker/n8n/data
N8N_ENV_FILE          = /home/kebron/git/mydocker/n8n/data/config

TRAEFIK_COMPOSE_DIR   = /home/kebron/git/mydocker
TRAEFIK_SERVICE_NAME  = traefik
TRAEFIK_NETWORK       = my_network

N8N_ENCRYPTION_KEY    = <DO NOT COPY HERE — lives in N8N_ENV_FILE on box>
```

SSH user `n8n-ops` is in `docker` group; key-based auth only; no password.
`N8N_ENCRYPTION_KEY` decrypts every credential stored in n8n — back it up
separately to Vaultwarden.

## When adding a new key

1. `.env.example` (commit) — empty entry under the right section
2. Your local `.env` — actual value
3. `env.config.json` — declare which projects need it
4. This file — add a row in the right table + Vaultwarden entry
5. Vaultwarden — store the value
6. `pnpm sync:env` to regenerate per-project files

All in one commit.
