# strguests.tools — SEO Audit

**Audit date:** 2026-05-10
**Auditor:** Claude (Opus 4.7)
**Scope:** Full source-grounded audit of `STRGuests-Tools/` (Astro 4.x + Tailwind static site, with planned serverless layer for Phase 3 AI generators).
**Live URL:** https://strguests.tools
**Production sitemap fetched:** yes (https://strguests.tools/sitemap-index.xml → sitemap-0.xml, 35 URLs)

---

## Executive summary

The site has a clean SEO foundation inherited from strhost.tools — `Layout.astro` emits per-page title/description/canonical/OG/Twitter/JSON-LD uniformly, `seo.ts` ships Organization + WebApplication + FAQPage + Article builders, an RSS feed with autodiscovery is live, programmatic /templates/[scenario] pages are indexed in the sitemap (26 of them), and Satori-generated OG images are live (verified `/og/index.png` and `/og/welcome-book.png` both 200 OK).

The ten findings below are roughly the same-shape problems strhost.tools shipped with two days ago. The lessons exist in the network; they have not yet been ported here.

The biggest, fixable problems:

1. **Three Phase 3 generator routes are linked from the homepage and footer but return live 404.** `/listing-description`, `/review-response`, `/guest-messages` are referenced in `src/data/tools.json`, rendered as link cards on `src/pages/index.astro`, and shown in the footer's Generators list (`Footer.astro:21-27`) — but no source files exist (`src/pages/listing-description.astro`, etc. all absent). Live confirmed: all three return HTTP 404. From every page, the footer crawls four broken internal links. (P0)
2. **Footer also links to `/privacy`, `/terms`, `/disclosures`, and `/get-the-pdf` — all four 404.** `Footer.astro:37, 44-46` references; live URLs return HTTP 404. The actual lead-magnet page is `/get-the-templates`, not `/get-the-pdf` — typo or stale carry-over from strhost.tools Footer template. (P0)
3. **No `/404` page exists.** `src/pages/404.astro` is absent. The host serves a generic 404 (live check on `/nonexistent-test-page-404` returns HTTP 404 with no branded body), missing the chance to retain users + crawlers who hit a stale URL. Identical to strhost.tools P1. (P1)
4. **`/feed.xml` is missing from the sitemap.** Live `sitemap-0.xml` lists 35 URLs (homepage, about, contact, blog index, 5 blog posts, 4 PDF generators, /get-the-templates, /templates/ index, 26 scenario pages). `/feed.xml` is not listed. Same defect as strhost.tools shipped with. (P1)
5. **Sitemap also missing the 3 Phase 3 routes** — fine while they 404, but when they ship, the sitemap config in `astro.config.mjs:8-17` will not auto-include them unless the `sitemap()` integration discovers them as new routes. Verify the integration picks them up post-deploy or add `customPages`. (P1)
6. **/templates/[scenario] thin-content risk.** All 26 scenario pages render the same components: header from JSON, `exampleInput`, `exampleOutput`, two related-template links, identical FAQ block, identical email capture, identical funnel band. Unique content per page is ~150–250 words. Identical to the strhost.tools lodging-tax problem (46/51 boilerplate pages). Google's near-duplicate cluster heuristic will likely suppress the long tail. (P0)
7. **`BreadcrumbList` schema is absent everywhere.** `src/lib/seo.ts` has no `buildBreadcrumb()` function. Templates pages especially benefit (`Home → Templates → Late checkout request`). Same defect as strhost.tools P1. (P1)
8. **Title↔H1 mismatch on homepage.** `<title>` = "STR Guests Tools — Free generators for hosts to delight guests" (62 chars), but `<h1>` = "Free tools for hosts to delight guests." Title says "generators," H1 says "tools." Tighten one to match. (P2)
9. **Tool page `<title>` length is short** — most tool pages render `${tool.name} — strguests.tools` (e.g., "Welcome Book Builder — strguests.tools" = 38 chars). Adding the primary keyword to the title (e.g., "Airbnb Welcome Book Template Builder — Free PDF · strguests.tools") would lift CTR materially. The pattern is correct; the source string is just too generic. (P1)
10. **Organization `sameAs[]` is empty.** `seo.ts:39-41` has the array placeholder but no entries. Google uses `sameAs` for entity disambiguation. At minimum should include `https://thestrledger.com` (parent org) and the three sister sites. (P2)

Headline score: **68/100** — solid technical scaffolding, but four sitewide broken-link defects + thin-content risk on programmatic pages drag it well below where it should land in a sprint. Reachable in a week: **86+**.

## Prioritized remediation roadmap

### Quick wins (≤1 hour each)

1. **Fix Footer broken links** — replace `/get-the-pdf` with `/get-the-templates` in `Footer.astro:37`; remove or stub `/privacy`, `/terms`, `/disclosures`. Eliminates 4 of 7 sitewide footer 404s.
2. **Gate the 3 Phase 3 tool entries** in `src/data/tools.json` with `comingSoon: true`; render disabled cards on `index.astro:88-110` and `Footer.astro:21-27`. Eliminates the remaining 3 sitewide footer 404s.
3. **Create `src/pages/404.astro`** with Layout + cross-links to working tools + /templates/ + /blog.
4. **Add `customPages` to sitemap** for `/feed.xml`. Edit `astro.config.mjs:16`.
5. **Fix homepage title↔H1 mismatch** — pick "generators" or "tools" consistently. ~2 lines.
6. **Add `seoTitle` field per tool** in `tools.json` and reference it from each tool page so titles include "Airbnb [keyword]". Material CTR win.
7. **Populate `sameAs[]`** in `seo.ts:39-41` with the 4 sister-site URLs at minimum.
8. **Decide trailing-slash convention** and fix `seo.ts:144-147` to match sitemap output.

### Medium (≤1 day each)

9. **Write 200-word MDX narrative per /templates/[scenario] page** (26 pages × ~7 min each = ~3 hrs writing). Wire `[scenario].astro` to render the MDX. Avoid the lodging-tax thin-content trap.
10. **Add `BreadcrumbList` JSON-LD** via `buildBreadcrumb()` in `seo.ts`; wire into `[scenario].astro`, `blog/[slug].astro`, /templates/index.astro.
11. **Add `ItemList` schema to /templates/index** for the 26 scenarios.
12. **Add `HowTo` schema** to each PDF tool page (each has a "How to use the printable" section perfect for HowTo).
13. **Create one-paragraph legal stubs** at `/privacy`, `/terms`, `/disclosures` and remove the temporary footer fix from #1.
14. **Add `<link rel="preload">` for critical fonts** + `font-display: swap` audit; add a static `/public/og-default.png` fallback referenced when generation fails.

### Strategic (≥1 day each)

15. **Ship Phase 3 (3 AI generators)** — unblocks `OPENAI_API_KEY` (or Claude Haiku 4.5 per spec §13). Removes 3 broken links AND opens 3 P2 keywords. Single highest-leverage strategic move.
16. **Pin program (Pinterest)** — run `scripts/build-pins.mjs`; ship 12+ pins per the keyword research doc. Pinterest is the parallel SEO channel for this cluster.
17. **Author profile + E-E-A-T** — add named human author (or "The STR Ledger team") for blog posts in `Article.author` instead of just `Organization`.
18. **GA4 + Search Console** — `Layout.astro:65-79` is gated on `PUBLIC_GA4_ID`; confirm it's set in production CI/CD. Cross-domain linker already wired across the four-site cluster + The STR Ledger (good).
