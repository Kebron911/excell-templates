# How to Use Your Guest Screening Log + Ban List

## What this workbook does

Per-booking screening checklist + portfolio-wide ban list. Records what you verified before accepting (verified ID, profile age, prior reviews, screening service used) and aggregates a "do not host" list across properties — so a problem guest banned at property A can't book property B.

## First-time setup (10 min)

1. **Property Roster** — list properties.
2. **Screening Settings** — your screening criteria (Airbnb verified ID required Y/N, minimum review count, minimum profile age, etc.) + which screening service you use (Autohost, RoomR, manual).
3. **Ban List** — pre-load any guests you've already banned across the portfolio.

## Every booking

1. New booking arrives. Open the **Per-Booking Checklist**:
   - Guest name + Airbnb / Vrbo profile URL
   - Verified ID? Y/N
   - Profile age (years since join)
   - Prior reviews (count + average rating)
   - Screening service result (PASS / WATCH / FAIL)
2. Cross-check against Ban List automatically (workbook flags if guest matches).
3. Accept / decline / request more info.

## Adding to the Ban List

When a guest violates rules (party, damage, harassment), add to the Ban List with:
- Guest name + profile URL
- Property where the incident happened
- Date
- Reason for ban (party / damage / harassment / fraud / other)
- Documentation links (photos, police report, Airbnb resolution case)

## Cross-property block detection

The Ban List feeds the Per-Booking Checklist as a lookup. When a banned guest tries to book any of your other properties, the checklist immediately flags them. The reason for ban + documentation links are right there for your decline message.

## Limitations — important

Airbnb / Vrbo limit how much guest data they expose pre-booking. You can't always see prior reviews on a new booking. Some "screening services" charge per-check ($1-5/booking) and pull deeper data. The workbook standardizes whatever data you have access to.
